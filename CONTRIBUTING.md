# Contributing

Contributions that improve passive, lawful OSINT investigation workflows are welcome.

## Scope

Accepted contributions include:

- parsers for public OSINT results
- canvas and evidence-management improvements
- accessibility, documentation and tests
- passive data sources with clear terms of service
- security hardening

Modules intended for phishing, credential theft, exploitation, persistence, evasion or unauthorized access are out of scope.

## Development

1. Fork the repository and create a focused branch.
2. Copy `.env.example` to `.env` and replace every development secret.
3. Run the validation commands documented in the README.
4. Use a Conventional Commit message.
5. Explain the user impact and testing in the pull request.
